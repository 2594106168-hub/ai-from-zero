🔌 API 详解

API是连接你和AI的管道。所有AI应用都建立在它上面。这一章把API讲到你不可能不懂。

1. 为什么需要API

你想让AI帮你写代码。但AI模型跑在云端的GPU服务器上——你不可能在自己笔记本上跑6710亿参数的模型。你只能发请求给云端服务器，让它算完把结果返回给你。这个"发请求→收结果"的通道就是API。

2. 用餐厅彻底搞懂

👤

你（客户端）

坐桌前想吃饭，不能进厨房。只能通过菜单和服务员。

📋

菜单（API文档）

写着能点什么菜、每道菜要什么配料。API文档就是菜单——告诉你有哪些功能可用、怎么调用。

🤵

服务员（API本身）

你告诉他我要麻婆豆腐，他去厨房下单，端回来。API就是服务员——在你的程序和AI之间传话。

👨‍🍳

厨房（AI服务器）

大厨根据订单做菜。大厨 = OpenAI/DeepSeek的大模型。

3. API Key不是会员，是预付费电卡

很多人以为API Key是买了个会员。不是的。API Key像预付费电表。你充值¥10，每次调用扣几分到几毛钱。余额为零就停。你不调用就不扣钱。这跟每月¥20的ChatGPT Plus完全不同。

OpenAI Key格式：sk-proj-xxxxx。Anthropic Key格式：sk-ant-xxxxx。DeepSeek Key格式：sk-xxxxx。拿到Key=能花你的钱=绝对不能泄露。

4. JSON — API的世界语

你的程序说Python话，AI服务器说C++话，中间怎么沟通？用JSON——极简单的纯文本格式，人机都能读。JSON只有6种类型：字符串、数字、布尔、null、数组、对象。

JSON — 这就是API的通信格式 
{
  "model": "deepseek-chat",         ← 用哪个模型
  "messages": [                         ← 对话内容
    { "role": "user", "content": "什么是API？" }
  ],
  "temperature": 0.7,                  ← 创意程度
  "max_tokens": 500                    ← 最多输出多少
}

这个JSON翻译成人话："喂DeepSeek，用deepseek-chat模型，温度0.7，最多输出500个token，帮我回答：什么是API？"

5. 请求-响应完整示例

你发的请求（Request） 
POST https://api.deepseek.com/v1/chat/completions
Content-Type: application/json
Authorization: Bearer sk-xxxxx      ← API Key 就放在这个头部！

{ "model": "deepseek-chat", "messages": [{"role":"user","content":"你好"}] }
AI返回的响应（Response） 
HTTP/1.1 200 OK
Content-Type: application/json

{
  "id": "chatcmpl-xxx",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "你好！有什么可以帮助你的吗？"
    }
  }],
  "usage": { "prompt_tokens": 8, "completion_tokens": 12, "total_tokens": 20 }
}

你看：你的代码构造JSON→发给API→API返回JSON→你的代码从choices[0].message.content拿到回复文字。

6. REST vs GraphQL vs gRPC

风格特点适合
RESTHTTP方法+名词URL，最主流绝大多数API
GraphQL前端指定要哪些字段，一个请求拿所有数据复杂前端应用
gRPCGoogle出品的二进制协议，超高性能微服务内部通信

你调的所有AI API都是REST风格。记住这个就够了。
