🚀 部署上线

你写的HTML文件，怎么变成全世界都能打开的网址？四种免费方案对比。

1. GitHub Pages — 最简免费方案

  1. 创建GitHub仓库（Public）
  2. 推送 index.html
  3. gh api repos/用户名/仓库/pages -X POST -f "source[branch]=master"
  4. 获得网址：https://用户名.github.io/仓库/

优点：永久免费、操作简单、全球CDN加速、自动HTTPS。国内直接能打开。局限：只能托管静态文件（HTML/CSS/JS/图片），不能跑后端。

2. Vercel — 纯静态+Serverless后端

Vercel既托管静态文件，也能跑后端代码（Serverless Functions）。你写一个 api/xxx.js 文件，Vercel自动把它部署成云端函数。免费额度：100GB带宽/月，100小时函数执行时长/月。个人项目足够。直接从 GitHub 导入仓库 → 自动部署 → 每次 push 自动更新。

注意：Vercel 默认域名 *.vercel.app 在国内有时不稳定，可绑自定义域名解决。

3. Cloudflare Pages & Workers

Cloudflare 的CDN在国内有节点，访问速度比Vercel快。Cloudflare Pages托管静态文件（免费、无限带宽）。Cloudflare Workers免费提供10万次/天Serverless函数执行。适合想要国内快速访问的项目。

需要先在 dash.cloudflare.com 注册workers.dev子域名，然后用 wrangler deploy 部署。

4. Deno Deploy

Deno Deploy 托管JS/TS服务端代码。免费额度每天10万次请求。部署极快（10秒上线）。支持直接连接GitHub仓库自动部署。

5. 方案选择

你的项目推荐
纯HTML/CSS/JS网站GitHub Pages
前端+需要后端APIVercel
国内用户为主Cloudflare Pages + Workers
JS/TS后端快速上线Deno Deploy
