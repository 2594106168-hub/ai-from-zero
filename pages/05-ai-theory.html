🧠 AI 理论

不要求你推导公式。但你需要知道Token/Temperature/ContextWindow，否则调不出好东西。

1. AI → ML → DL → LLM → Agent

AI(人工智能): 让机器表现出智能的一切技术 — 下棋、识图、翻译、开车
  └── ML(机器学习): 不靠人工写规则，让机器从数据中自动找规律
       └── DL(深度学习): 多层神经网络模拟人脑结构
            └── LLM(大语言模型): 海量文本训练出的"写文章机器"— GPT/Claude/DeepSeek
                 └── Agent(智能体): LLM + 工具调用 = 能自主执行任务的AI

2. LLM的本质：超强"下一个词预测器"

不要神话大模型。它做的基本上就是：给定前文，猜下一个最可能的词。然后一个词一个词续写下去，直到写完。它没有意识、不懂意义——但它见过万亿级别的文字，对词与词之间的概率关系掌握到极致。这种续写能力强大到让它看起来像在思考。

你输入"床前明月光，疑似地上"→它预测下一个词是"霜"→继续→输出"举头望明月，低头思故乡"。写代码、翻译、分析——全是同样的"续写"机制。

3. Token — 模型不是按字计费

1 Token ≈ 0.75个英文单词 ≈ 0.5个中文字。"Hello world"≈3 Token，"你好世界"≈4 Token。一段1000字的文章≈500-700 Token。100行Python≈200-400 Token。

每次API调用的费用 = (输入Token数+输出Token数) × 单价。DeepSeek单价每百万Token约¥2-8。

💰成本实测：问DeepSeek"写一首编程打油诗"→输入约15Token，输出约120Token，共135Token。按每百万Token ¥2算：135/1000000×2≈¥0.00027。不到0.03分钱。充¥10能聊几万次。

4. Temperature — 控制AI的"创意浓度"

范围0~2。T=0 → 模型只选概率最高的词。稳定、可重复。适合：代码、事实回答、翻译。T=1 → 有一定随机性。有创意但可能不准确。适合：写作、故事。T>1 → 开始胡说。

🎯经验公式：代码/数据→0.0~0.3。文章/邮件→0.7~0.9。头脑风暴→1.0。永远不要超过1.5。

5. Context Window — 一次能"读"多少

Context Window = 模型一次能看多少字。GPT-4o是128K(≈300页)，Claude是200K(≈中篇小说)，DeepSeek V4是1M(≈《三体》第一部)。Context越大越贵——你放进去的所有内容都计费。

实际开发中：Context Window ≥ 你的整个项目代码，才能让AI理解全貌。如果不够，就需要RAG（第14章）——先检索相关片段再喂给AI。

6. System Prompt vs User Prompt

System Prompt = 给AI预设角色+规则（"你是资深Python工程师，用中文+emoji回复"）。不算在对话历史里，不影响计费。User Prompt = 用户的实际问题。你可以把System Prompt想成AI的"人设"——它贯穿整个对话，AI一直知道自己扮演什么角色。

7. 训练 vs 推理

训练（Training）：用海量数据+天价GPU训练出一个模型。你可能没见过这个过程。推理（Inference）：你调用API，模型跑一次计算、返回结果。你每次调API都是在做推理。训练费几千万美元；推理费几厘钱。
