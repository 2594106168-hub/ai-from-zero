🌐 计算机网络

你在浏览器里点了一下，数据怎么从你的电脑跑到地球另一边的服务器再跑回来？这一章全部讲清楚。

1. 一个消息的环球旅行

用微信给朋友发了"你好"。这两个字走了多远？

你的手机 → WiFi路由器 → 光猫 → 运营商基站 → 光纤 → 微信服务器 → 朋友的手机
  (30ms)     (0.1ms)       (5ms)     (10ms)        (15ms)

每一步，你的数据都以电磁波或光信号的形式在网线上传输。到了目的地再还原成文字。整个过程像快递——打包→贴地址→运输→拆包→签收。你按了发送，三十几毫秒后对方就看到了。

2. IP 地址 — 互联网门牌号

每台联网的设备都有一个唯一的 IP 地址。就像你家有门牌号、你有手机号。没有 IP 地址，别的机器找不到你。

IPv4（老标准）：用 4 个数字，每个 0-255。比如 Google 的 142.250.72.46。全球最多约 43 亿个。不够用了，所以有了 IPv6（新标准）——用六组十六进制数（如 2001:db8::1），多到地球上每粒沙子都能分一个。

你现在用的 IP 是 局域网 IP（如 192.168.1.5）——路由器分配给你的内部编号。路由器对外只有一个公网 IP。这叫 NAT（网络地址转换）——一台路由器背后可以藏几十台设备，共用一个公网 IP。

3. DNS — 互联网的电话簿

你记得 google.com，但记不住 142.250.72.46。DNS（Domain Name System）就是把网址翻译成IP的全球分布式电话簿。

你在浏览器打 "google.com"
         ↓
 [浏览器DNS缓存] 有记录吗？ → 没有。继续问。
         ↓
 [操作系统DNS缓存] 有吗？ → 没有。继续问。
         ↓
 [你的路由器] 有吗？ → 没有。继续问。
         ↓
 [运营商DNS服务器] 有记录吗？ → 有！返回 142.250.72.46
         ↓
 浏览器向这个IP发请求 → 页面显示

DNS 劫持：恶意程序改掉你的 DNS 设置，把 google.com 翻译到色情网站。你之前 WiFi 跳黄网就是这个原因。换成干净DNS：阿里 223.5.5.5、Cloudflare 1.1.1.1、Google 8.8.8.8。

4. 端口 — 一台机器跑几十个服务

IP 找到哪台机器。端口找到这台机器上的哪个服务。

类比：IP = 写字楼的地址。端口 = 楼里的房间号。你寄快递要写"XX大厦801室"。发 HTTP 请求就是"IP + 端口80"。

端口标准服务你什么时候用到
80HTTP（明文网页）每次打开 http://xxx
443HTTPS（加密网页）地址栏有🔒时
22SSH（远程登录服务器）部署网站
3306MySQL 默认端口开数据库
3000开发服务器默认端口npm run dev / node server.js

5. HTTP 协议 — 双方怎么对话

IP 找到机器、端口找到服务。剩下就是「怎么说话」——用 HTTP 协议。

HTTP 请求必须包含：①用什么方法（GET/POST…）②找哪个资源（路径 /index.html）③协议版本（HTTP/1.1）④Host 头部（找哪个网站——因为一台服务器可能托管几百个网站）。

HTTP 请求 — 浏览器发给服务器 
GET /api/users HTTP/1.1                   ← 方法 路径 版本
Host: www.example.com                      ← 找哪个网站
Authorization: Bearer sk-xxxx              ← 身份认证（API Key 在这！）
Content-Type: application/json             ← 请求体是什么格式
Accept: application/json                   ← 我想要什么格式的返回
HTTP 响应 — 服务器返回给浏览器 
HTTP/1.1 200 OK                            ← 200=成功，最重要的状态码
Content-Type: application/json; charset=utf-8
Content-Length: 256                        ← 返回内容的大小

{ "name": "张三", "age": 25 }              ← 真正的返回数据（JSON）

6. HTTP 方法 — GET/POST/PUT/DELETE

方法干什么类比有请求体吗幂等吗
GET读取数据淘宝搜"鞋子"→看到结果❌ 没有✅ 是
POST创建数据注册账号→填表提交✅ 有❌ 不
PUT完整替换把整篇文章替换成新版本✅ 有✅ 是
PATCH部分更新只改文章标题，正文不动✅ 有❌ 不
DELETE删除删帖子❌ 没有✅ 是
💡幂等是什么意思？同一个请求，发1次和发100次结果一样=幂等。GET 查数据，查100次都一样=幂等。POST 创建用户，创建100次就有100个用户=不幂等。PUT 替换，第100次替换和第1次结果一样=幂等。

GET 和 POST 最核心的区别：GET 把参数放在网址后面（?name=张三&age=25），POST 把参数藏在请求体里。所以密码绝对不能用 GET——网址会被别人看到、被浏览器历史记录。用 POST。

7. HTTP 状态码

2xx 成功了

200 OK — 正常
201 Created — 创建成功
204 No Content — 成功但没数据返回（删除常用）

3xx 重定向

301 — 永久搬家，以后直接去新地址
302 — 临时搬家
304 — 你的缓存没过期，别重复发

4xx 你的锅

400 Bad Request — 请求格式不对
401 Unauthorized — 没登录或Key过期
403 Forbidden — 你没权限
404 Not Found — 这个路径不存在
429 Too Many — 请求太频繁被限速

5xx 服务器锅

500 Internal Error — 服务器崩了
502 Bad Gateway — 网关挂了
503 Service Unavailable — 服务器在维护

8. HTTPS 为什么比 HTTP 安全

HTTP 传输是明文的。任何人截获数据包都能直接看到你发的密码、银行卡号。HTTPS = HTTP + TLS 加密。数据在发送前加密，中间截获的是一堆乱码，只有服务器有密钥能解开。地址栏有🔒=HTTPS。现在几乎所有网站都用 HTTPS。

9. REST API — 最主流的 API 设计风格

REST 不是技术，是一套命名规范。用 HTTP 方法+名词化的 URL 来操作资源：

REST API 设计 
# 获取所有用户    GET    /api/users
# 获取ID=1的用户   GET    /api/users/1
# 创建用户         POST   /api/users       (+ 请求体含用户信息JSON)
# 完整替换用户     PUT    /api/users/1      (+ 请求体含所有字段)
# 只改年龄         PATCH  /api/users/1      (+ 请求体只含age字段)
# 删除用户         DELETE /api/users/1

这就是你调用 DeepSeek API 的本质：POST 一个 JSON 到 https://api.deepseek.com/v1/chat/completions → 服务器处理 → 返回 JSON 结果。

10. WebSocket — 不只是请求-响应

HTTP 是"你问我才答"。但聊天应用、实时股票——服务器需要主动推给你数据。WebSocket 就解决这个问题：建立一条长连接，服务器随时推数据。Claude Code 和 Codex 的流式输出底层用的就是类似机制。
