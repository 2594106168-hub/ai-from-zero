🗄️ 数据库

你的数据存哪里、怎么查。SQL vs NoSQL。新手从SQLite开始。

1. SQL（关系型）vs NoSQL（非关系型）

📊

SQL

数据像Excel表，有固定的列结构。表与表之间有关联（外键）。MySQL、PostgreSQL、SQLite。用SQL语言查询。适合：用户表、订单表、任何结构固定的数据。

📦

NoSQL

数据像JSON文档，结构灵活。MongoDB、Redis、Firebase。各有各的查询方式，不统一。适合：日志、缓存、用户行为、结构不固定的数据。

2. 核心SQL语句

SQL 
SELECT * FROM users WHERE age > 18;                    -- 查：找所有成年用户
INSERT INTO users (name, age) VALUES ('张三', 25);     -- 增：创建用户
UPDATE users SET age = 26 WHERE name = '张三';         -- 改：更新年龄
DELETE FROM users WHERE id = 1;                        -- 删：删除用户
-- JOIN：关联查询（两张表关联）
SELECT u.name, o.total FROM users u JOIN orders o ON u.id = o.user_id;

3. SQLite — 一行代码建数据库

SQLite不需要装服务器——一个文件就是整个数据库。Python自带支持。个人项目、原型、小工具，SQLite完全够用。

Python + SQLite 
import sqlite3
conn = sqlite3.connect("myapp.db")  # 文件即数据库！
c = conn.cursor()
c.execute("""CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    age INTEGER,
    email TEXT UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)""")
c.execute("INSERT INTO users (name,age) VALUES (?,?)", ("张三",25))
rows = c.execute("SELECT * FROM users WHERE age > ?", (18,)).fetchall()
conn.commit(); conn.close()

4. 什么时候该升级

SQLite → PostgreSQL 的信号：①多用户同时写入（并发）②数据量超过10万条且需要复杂查询 ③需要高可用（主从复制、自动故障转移）④需要更丰富的数据类型（地理坐标JSONB、全文搜索等）。自己用的工具站 SQLite 够用到天荒地老。
