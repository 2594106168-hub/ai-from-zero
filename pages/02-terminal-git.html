⌨️ 终端 & Git

黑框框不可怕。10个命令+Git快照原理+GitHub Pages部署。

1. 终端为什么重要

很多编程工具没有图形界面，只能在终端用。Claude Code、Codex CLI、npm、pip——全跑在终端里。你不是要成为命令行大师，但必须能在终端里：①打开正确的文件夹 ②运行一个命令 ③看懂输出。

你怎么打开终端：Windows → Win+R → 输入 powershell → 回车。macOS → 聚焦搜索 → 输入 terminal → 回车。

2. 10个必记命令

终端 
ls                # 列出当前文件夹所有文件 (macOS/Linux)
dir               # 同上 (Windows)
cd 文件夹名         # 进入这个文件夹
cd ..              # 回上一级
mkdir 新文件夹      # 创建文件夹
pwd               # 显示当前完整路径 (macOS/Linux)
node 文件.js        # 运行JavaScript
python 文件.py      # 运行Python
npm install 包名    # 安装Node.js包
code .             # 用 VS Code 打开当前文件夹

3. Git 解决什么问题

你写了一个网站。第二天改50行代码。第三天发现改坏了——但已经不知道原来什么样了。Git 就是解决这个的。

每次你执行 git commit，Git 就给整个项目拍一张快照。任何时刻改坏了，随时回到任意一张快照。你还可以创建分支——从主线分一条支线，在上面随便实验。做好了合并回去，做烂了删掉分支，主线毫发无损。GitHub = 把快照存在云上。

4. Git 核心概念：工作区 → 暂存区 → 仓库

工作区（你正在改的文件）→ git add → 暂存区（列入下次提交清单）→ git commit → 本地仓库（拍快照）→ git push → 远程仓库（GitHub）

这个三步流是理解 Git 的关键：你改了文件（工作区）→ git add 选择哪些改动要提交（暂存区）→ git commit 拍快照（仓库）。

5. 每天做的就这5步

终端 
git status              # 看哪些文件被改了
git add .               # 暂存所有改动
git commit -m "修复了首页bug"  # 提交（-m后面写备注）
git push                # 推到 GitHub
git pull                # 拉取别人的更新

6. 分支 — 做实验不搞坏主线

终端 
git branch              # 看所有分支
git checkout -b 新功能   # 创建并切换到新分支
# ...在新分支上改代码、提交...
git checkout master     # 切回主线
git merge 新功能         # 把支线的改动合并进来
git branch -d 新功能     # 删掉不用的分支

7. .gitignore — 防止 API Key 泄露

.gitignore 
node_modules/       # npm安装的依赖包，体积巨大
.env                # 存API Key的文件，绝对不能提交！
__pycache__/        # Python缓存
*.pyc               # 编译文件
dist/ build/        # 构建产物
⚠️如果 API Key 已经push到了 GitHub：这密钥已经公开了。①去平台后台立即删除这个 Key 重新生成 ②在代码里替换新 Key ③用 git commit --amend 或强制推送覆盖历史。不是改个文件就行的——Git 历史里还有旧版本。

8. GitHub Pages — 免费让你的代码变成网站

  1. 创建 GitHub 仓库 → Public
  2. 把 index.html 推上去
  3. 终端运行：gh api repos/用户名/仓库名/pages -X POST -f "source[branch]=master"
  4. 得到网址：https://用户名.github.io/仓库名/

全世界都能打开。不需要买服务器。这是个人开发者最常用的免费部署方案。
