🤖 AI Agent

普通LLM问你一句答一句。Agent给你目标——自己规划、用工具、循环执行直到完成。

1. Agent是什么

普通LLM：你问"这段代码有bug吗"→它说"第3行少了分号"→你自己去改。Agent：你说"检查这个项目有没有bug，有就修了"→它自己读所有文件、找到3个bug、逐一修复、告诉你修了哪些。

Agent = LLM + 工具调用 + 循环决策

2. 工作循环：ReAct模式

  1. 思考（Thought）：我现在该做什么？有哪些工具可用？
  2. 行动（Action）：调用一个工具——搜索API、读文件、写代码、执行命令
  3. 观察（Observation）：工具返回了什么？
  4. 回到步骤1，直到目标达成
用户: "帮我做市场调研：上周AI行业有什么大新闻？整理成报告发邮件给我。"
  ↓
Agent: 1) 调用web_search搜索"AI news last week"
      2) 阅读搜索结果→提取关键信息→调用write_file写报告
      3) 调用send_email发送→"✅已发送，请查收邮件"

3. Function Calling — Agent的底层机制

在第7章里你看到了Function Calling的基础写法。Agent就是把它放大：给AI十几个工具（查网页、写文件、发邮件、读数据库、搜索代码…），AI自己决定用哪个、怎么组合。

Python — Agent循环伪代码 
def agent_loop(goal, tools):
    msgs = [{"role":"system", "content": f"你是一个Agent。目标:{goal}。可用工具:{tools}"}]
    while True:
        r = client.chat.completions.create(model="deepseek-chat", messages=msgs, tools=tools)
        if r.choices[0].finish_reason == "stop": return r  # 完成了
        # AI要求调用工具 → 执行 → 结果塞回对话 → 继续
        tool_call = r.choices[0].message.tool_calls[0]
        result = execute_tool(tool_call)
        msgs.append({"role":"tool", "content": result})

4. MCP（Model Context Protocol）

MCP是Anthropic推出的AI与工具的连接标准。类比：USB统一了电脑和键盘/鼠标/U盘的连接方式，MCP统一了AI和各类外部工具（数据库、文件系统、第三方API）的连接方式。

只要一个工具支持MCP，任何支持MCP的AI程序都能直接调用。不用为每个AI重写适配代码。Claude Code、Cursor、OpenClaw都已支持MCP。官方文档
