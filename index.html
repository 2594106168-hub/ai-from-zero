第 1 章

💻 计算机基础 — 从零开始

如果你连「终端」和「文件」都不太清楚，这一章就是为你准备的。

1.1 计算机是怎么工作的

🧠

CPU（中央处理器）

计算机的大脑。执行指令、做计算。主频越高越快，核心越多越能同时干多件事。

💾

内存（RAM）

CPU 的工作台。数据从硬盘加载到内存，CPU 直接在内存上读写。断电后数据消失。

💿

硬盘（SSD/HDD）

永久存储数据的地方。你的文件、程序都在这。SSD 比 HDD 快 10 倍以上。

🖥️

操作系统（OS）

管理硬件和软件的大管家。Windows、macOS、Linux。你所有的操作都是通过它完成的。

💡一句话理解：硬盘存东西 → 点开程序 → 加载到内存 → CPU 执行 → 显示在屏幕上。

1.2 文件系统 — 你电脑里的一切都是文件

计算机用文件夹（目录）组织文件。每个文件有路径：

操作系统路径示例分隔符
WindowsC:\用户\张三\桌面\简历.docx\
macOS / Linux/home/zhangsan/Desktop/photo.jpg/
📁

文件后缀名

.html 是网页、.py 是 Python 程序、.js 是 JavaScript、.json 是数据文件。后缀名告诉电脑该用什么程序打开。

🔗

相对路径 vs 绝对路径

绝对路径=从根目录开始的完整路径。相对路径=相对于当前目录的路径。./data/info.json 表示当前目录下的 data 文件夹里的文件。

终端（Terminal）就是那个黑框框。你不需要成为命令行大师，但你需要知道怎么打开它、怎么在里面跑命令。

1.4 互联网是怎么连接世界的

🌐

IP 地址

每台联网设备的唯一地址，像快递的门牌号。IPv4 如 192.168.1.1，IPv6 如 fe80::1。

🔤

域名 (Domain)

IP 地址太难记，所以有了域名。你在浏览器输入 google.com，DNS 服务器自动翻译成 IP 地址。

📡

HTTP / HTTPS

浏览器和服务器之间的通信协议。HTTPS 是加密版（有🔒标志），HTTP 是明文版。

🏠

服务器 (Server)

一台 24 小时开机的远程电脑。你访问的网站、用的 APP，数据全在服务器上。你的电脑是「客户端」。

第 3 章

🔌 什么是 API — 你连接 AI 的管道

API 是你使用一切 AI 工具的基础。搞懂它，后面的路全是平的。

3.1 用「餐厅点菜」完全理解 API

👤

你（客户端）

你坐在桌前想吃饭，但不能进厨房。只能通过菜单和服务员点菜。

📋

菜单（API 文档）

写着能点什么菜、每道菜需要什么配料。API 文档就是菜单——告诉你有哪些功能可用。

🤵

服务员（API）

你告诉服务员要什么，服务员去厨房下单，然后把菜端回来。API 就是服务员——在你的程序和 AI 之间传话。

👨‍🍳

厨房（AI 服务器）

大厨根据订单做菜。大厨就是 OpenAI/DeepSeek 的大模型，负责「思考」和「生成」内容。

3.2 请求（Request）和 响应（Response）

整个过程就两个东西：

  • 请求 (Request)：你对服务员说「我要一份麻婆豆腐」→ 你的程序向 API 发送一句话「帮我写段代码」
  • 响应 (Response)：服务员把麻婆豆腐端回来 → API 把 AI 生成的结果返回给你的程序

3.3 API Key — 你的门禁卡

API Key 就是你的门禁卡。每次你让服务员去厨房，服务员都要刷你的卡，厨房才认。没有 API Key，厨房不理你。

⚠️API Key 是私密信息！不能发到 GitHub、不能截图发朋友圈。OpenAI 的 Key 以 sk- 开头，DeepSeek 的也是 sk- 开头。别人拿到你的 Key 就能以你的名义花钱调用 API。

3.4 JSON — API 的通用语言

API 之间用 JSON（JavaScript Object Notation）交换数据。它是最简单、最通用的数据格式。

JSON 
{
  "name": "张三",
  "age": 25,
  "skills": ["Python", "JavaScript"],
  "contact": {
    "email": "zhangsan@example.com",
    "phone": "13800000000"
  }
}

你看，这就是键值对的嵌套。AI 返回的结果、你发送的问题，全用 JSON 包装。

4.2 大语言模型（LLM）到底在干什么

一句话说清楚：LLM 就是一个「超强续写机器」。你给它一段文字，它预测下一个最可能的词，然后一个个续写下去。

🎯例子：你输入「床前明月光，疑似地上」→ 它预测下一个词是「霜」→ 继续预测下一句 → 输出整首诗。对代码、翻译、写作都是同样的原理——输入 + 预测 = 输出。

4.3 Token（令牌）— 模型的计价单位

模型不是按「字」算钱，是按 Token 算。1 个 Token ≈ 0.75 个英文单词 ≈ 0.5 个中文字。

文本大约 Token 数
"Hello world"~3
"你好世界"~4
一段 1000 字的中文文章~500-700
一段 100 行的 Python 代码~200-400
💰用 DeepSeek API 的话，1000 个 Token 大概花 几厘钱。你充 10 块钱可以玩好几个月。

4.4 模型的关键参数

🌡️

Temperature（温度）

控制输出的随机性。0=稳定可预测，1=有创意但可能胡说。写代码用 0~0.3，写文章用 0.7~1。

📏

Max Tokens

限制模型最多输出多少 Token。防止它写个没完。一般设置为 1000-4000。

📐

Context Window（上下文窗口）

模型一次能「看到」多少内容。GPT-4o 是 128K，Claude 是 200K，DeepSeek V4 是 1M——相当于一次读完一本厚厚的书。

⚙️

System Prompt（系统提示）

给模型预设的角色和行为。比如「你是一个编程老师，用中文回答」。这个不作为用户输入计费。

第 6 章

⚡ API 实操 — 第一行代码就让 AI 为你工作

安装库 → 获取 Key → 写代码 → AI 开始回复。10 分钟跑通。

6.1 用 DeepSeek API（推荐新手）

  1. 去 platform.deepseek.com 注册 → 短信验证 → 充值 ¥10
  2. 左侧「API Keys」→ 创建 → 复制 Key（sk- 开头，只显示一次）
  3. 终端安装依赖：pip install openai
  4. 运行下面的代码：
Python — 第一次调用 DeepSeek 
from openai import OpenAI

client = OpenAI(
    api_key="sk-你的Key放这里",
    base_url="https://api.deepseek.com/v1"
)

response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "system", "content": "你是中文编程老师，用简单的例子解释"},
        {"role": "user", "content": "用一句话解释什么是递归"}
    ]
)

print(response.choices[0].message.content)
✅跑通了？你已经完成了最重要的一步。接下来所有复杂的 AI 应用，本质上都是这段代码的扩展。

6.2 用 Claude API

Python 
from anthropic import Anthropic

client = Anthropic(api_key="sk-ant-你的Key")

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1000,
    system="你是中文编程老师",
    messages=[{"role": "user", "content": "解释什么是异步编程"}]
)

print(message.content[0].text)

6.3 多轮对话

把之前的对话历史一起发回去，模型就能「记住」上下文：

Python 
conversation = [
    {"role": "system", "content": "你是日语老师"},
    {"role": "user", "content": "「ありがとう」是什么意思？"},
    {"role": "assistant", "content": "「ありがとう」是"谢谢"的意思，用于表达感谢。"},
    {"role": "user", "content": "那再告诉我一个类似的礼貌用语"}
]

response = client.chat.completions.create(
    model="deepseek-chat",
    messages=conversation
)

6.4 流式输出（打字机效果）

Python 
stream = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": "写一首关于编程的打油诗"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")

第 8 章

🎯 提示词工程 — 让 AI 听话的艺术

同样的模型，提示词好坏差 10 倍效果。这是 AI 时代的核心技能。

8.1 提示词万能公式

  角色 + 任务 + 约束 + 格式 = 完美提示词

  角色：你是一个 XX 专家
  任务：请帮我做 XX
  约束：要求 XX，不要 XX
  格式：用 XX 格式输出

8.2 好提示词 vs 烂提示词

❌ 烂

"帮我写个网站"

AI 不知道你要什么网站、什么颜色、什么功能，出来的东西大概率不是你要的。

✅ 好

"你是一个资深前端工程师。帮我做一个外卖点餐网站的首页，目标用户是大学生。要求：橙色+白色配色、横向滑动热门餐厅卡片、顶部搜索栏、底部4个导航图标。用 HTML+CSS+JS 单文件实现，移动端优先，代码加中文注释。"

8.3 常用提示词技巧

技巧说明示例
角色扮演给 AI 设定专家身份"你是一个 10 年经验的 Python 后端架构师"
少样本学习给它看几个例子再让它做"输入:hello → 输出:你好。输入:goodbye → 输出:再见。输入:thanks → "
思维链让它一步步推理"先分析问题，列出步骤，再逐步执行"
拆分任务复杂任务拆成多个小步"先写 HTML 结构，我确认了再写 CSS"
输出格式指定要什么格式"用 JSON 格式返回，包含 name、age、skills 三个字段"
约束输出控制不要什么"不要引入任何第三方库，用纯原生实现"

8.4 遇到报错怎么办

🔄黄金循环：说需求 → AI 写代码 → 运行 → 报错了 → 把报错信息完整贴给 AI → AI 修 → 再运行 → 成功。你不需要懂报错是什么意思，AI 懂就行。

9.3 项目三：Python 数据爬虫（1-3小时）

技术：Python + requests + BeautifulSoup
定价：¥100-500
怎么做：告诉 Claude Code 要爬什么网站、取什么数据 → 导出 CSV/Excel → 发给客户

Python — 爬虫骨架 
import requests
from bs4 import BeautifulSoup
import csv

url = "https://example.com"
resp = requests.get(url)
soup = BeautifulSoup(resp.text, "html.parser")

# 提取数据（让 Claude Code 帮你写具体的选择器）
data = []
for item in soup.select(".item"):
    data.append({
        "title": item.find("h2").text,
        "price": item.find(".price").text
    })

# 导出 CSV
with open("output.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=data[0].keys())
    writer.writeheader()
    writer.writerows(data)
